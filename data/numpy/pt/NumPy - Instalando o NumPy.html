Instalando o NumPy

Dica

Esta página assume que você está confortável usando um terminal e está familiarizado com gerenciadores de pacotes. O único pré-requisito para instalar o NumPy é o próprio Python. Se não tiver Ainda Python e querendo a forma mais simples de começar, recomendamos que você use a Distribuição Anaconda - ela inclui Python, NumPy e muitos outros pacotes comumente usados para computação científica e ciência de dados.

O método recomendado para instalar o NumPy depende do seu fluxo de trabalho preferido. Abaixo, dividimos os métodos de instalação nas seguintes categorias:

  • Baseado em projetos (ex.: UV, Pixi) (recomendado para novos usuários)
  • Baseado em ambiente (por exemplo, pip, conda) (o fluxo de trabalho tradicional)
  • Gerenciadores de pacotes de sistema (não recomendados para a maioria dos usuários)
  • Construindo a partir do código-fonte (para usuários avançados e fins de desenvolvimento)

Escolha o método que melhor atende às suas necessidades. Se você estiver em dúvida, comece pelo método baseado em ambiente usando ou .condapip

Abaixo estão os diferentes métodos para instalar o NumPy. Clique nas abas para explorar cada método:

Recomendado para novos usuários que querem um fluxo de trabalho mais ágil.

  • UV: Um gerenciador de pacotes moderno em Python projetado para velocidade e simplicidade.

    uv pip install numpy

  • Pixi: Um gerenciador de pacotes multiplataforma para Python e outras linguagens.

    pixi add numpy

As duas principais ferramentas que instalam pacotes em Python são e . A funcionalidade deles se sobrepõe parcialmente (por exemplo, ambos podem ser instalados), porém, eles também podem funcionar juntos. Vamos discutir as principais diferenças entre pip e conda aqui – isso é importante de entender se você quiser gerenciar pacotes de forma eficaz.pipcondanumpy

A primeira diferença é que o conda é multilinguagem e pode instalar Python, enquanto o pip é instalado para um Python específico no seu sistema e instala outros pacotes apenas nessa mesma instalação de Python. Isso também significa que o Conda pode instalar bibliotecas e ferramentas que não sejam Python que você possa precisar (por exemplo, compiladores, CUDA, HDF5), enquanto o pip não pode.

A segunda diferença é que o pip instala a partir do Python Packaging Index (PyPI), enquanto o conda instala por seus próprios canais (tipicamente "defaults" ou "conda-forge"). O PyPI é de longe a maior coleção de pacotes, porém, todos os pacotes populares também estão disponíveis para a Conda.

A terceira diferença é que o conda é uma solução integrada para gerenciar pacotes, dependências e ambientes, enquanto com pip você pode precisar de outra ferramenta (existem muitas!) para lidar com ambientes ou dependências complexas.

  • Conda: Se você usa o conda, pode instalar o NumPy pelos canais padrão ou conda-forge:
    conda create -n my-env
conda activate my-env
conda install numpy
  • Pip:
    pip install numpy

Tip: Use a virtual environment for better dependency management

python -m venv my-env
source my-env/bin/activate  # macOS/Linux
my-env\Scripts\activate     # Windows
pip install numpy

Not recommended for most users, but available for convenience.

macOS (Homebrew):

brew install numpy

Linux (APT):

sudo apt install python3-numpy

Windows (Chocolatey):

choco install numpy

For advanced users and developers who want to customize or debug NumPy.

A word of warning: building Numpy from source can be a nontrivial exercise. We recommend using binaries instead if those are available for your platform via one of the above methods. For details on how to build from source, see the building from source guide in the Numpy docs.

Verificando a Instalação#

Após instalar o NumPy, verifique a instalação executando o seguinte em um shell ou script em Python:

import numpy as np
print(np.__version__)

Isso deve imprimir a versão instalada do NumPy sem erros.
