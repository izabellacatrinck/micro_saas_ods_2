O que é NumPy?#

O NumPy é o pacote fundamental para computação científica em Python. É uma biblioteca Python que fornece um objeto array multidimensional, vários objetos derivados (como matrizes e arrays mascarados), e um variedade de rotinas para operações rápidas em matrizes, incluindo matemática, lógica, manipulação de formas, ordenação, seleção, E/S, transformadas de Fourier discretas, álgebra linear básica, estatística básica operações, simulação aleatória e muito mais.

No núcleo do pacote NumPy está o objeto. Isso encapsula matrizes n-dimensionais de tipos de dados homogêneos, com muitas operações são realizadas em código compilado para desempenho de desempenho. Existem várias diferenças importantes entre os arrays NumPy e o sequências padrão em Python:

  • Arrays NumPy têm tamanho fixo na criação, ao contrário das listas Python (que pode crescer dinamicamente). Alterando o tamanho de um testamento Crie um novo array e exclua o original.

  • Os elementos em um array NumPy precisam ser todos iguais tipo de dado, e portanto terá o mesmo tamanho na memória. A exceção: pode-se ter arrays de objetos (Python, incluindo NumPy), assim permitindo arranjos de elementos de diferentes tamanhos.

  • Matrizes NumPy facilitam matemáticas avançadas e outros tipos de Operações em grandes quantidades de dados. Normalmente, tais operações são executado de forma mais eficiente e com menos código do que seria possível usando Sequências embutidas do Python.

  • Uma crescente profusão de pessoas científicas e matemáticas baseadas em Python os pacotes usam arrays NumPy; embora esses normalmente apoiem Entrada de sequência em Python, eles convertem essa entrada em arrays NumPy antes para processamento, e frequentemente geram arrays NumPy. Em outras palavras, para usar eficientemente grande parte (talvez até a maioria) dos modelos atuais software científico/matemático baseado em Python, só sabendo como usar os tipos de sequência embutidos do Python é insuficiente - também um precisa saber usar arrays NumPy.

Os pontos sobre tamanho e velocidade da sequência são particularmente importantes em computação científica. Como exemplo simples, considere o caso de multiplicando cada elemento em uma sequência 1D pelo correspondente elemento em outra sequência do mesmo comprimento. Se os dados são armazenado em duas listas Python, e , poderíamos iterar sobre Cada elemento:ab

c = []
for i in range(len(a)):
    c.append(a[i]*b[i])

Isso produz a resposta correta, mas se e cada contêm milhões de números, pagaremos o preço pelas ineficiências de fazendo loop em Python. Poderíamos realizar a mesma tarefa muito mais rapidamente em C escrevendo (para maior clareza, negligenciamos declarações de variáveis e inicializações, alocação de memória, etc.)ab

for (i = 0; i < rows; i++) {
  c[i] = a[i]*b[i];
}

Isso economiza toda a sobrecarga envolvida na interpretação do código Python e manipular objetos Python, mas às custas dos benefícios adquirido programando em Python. Além disso, o trabalho de codificação exigido aumenta com a dimensionalidade dos nossos dados. No caso de um 2D array, por exemplo, o código C (abreviado como antes) expande-se para

for (i = 0; i < rows; i++) {
  for (j = 0; j < columns; j++) {
    c[i][j] = a[i][j]*b[i][j];
  }
}

O NumPy nos oferece o melhor dos dois mundos: operações elemento por elemento são o "modo padrão" quando um está envolvido, mas o A operação elemento por elemento é executada rapidamente por C pré-compilado código. No NumPy

c = a * b

does what the earlier examples do, at near-C speeds, but with the code simplicity we expect from something based on Python. Indeed, the NumPy idiom is even simpler! This last example illustrates two of NumPy’s features which are the basis of much of its power: vectorization and broadcasting.

Why is NumPy fast?#

Vectorization describes the absence of any explicit looping, indexing, etc., in the code - these things are taking place, of course, just “behind the scenes” in optimized, pre-compiled C code. Vectorized code has many advantages, among which are:

  • vectorized code is more concise and easier to read

  • fewer lines of code generally means fewer bugs

  • the code more closely resembles standard mathematical notation (making it easier, typically, to correctly code mathematical constructs)

  • vectorization results in more “Pythonic” code. Without vectorization, our code would be littered with inefficient and difficult to read loops.for

Broadcasting is the term used to describe the implicit element-by-element behavior of operations; generally speaking, in NumPy all operations, not just arithmetic operations, but logical, bit-wise, functional, etc., behave in this implicit element-by-element fashion, i.e., they broadcast. Moreover, in the example above, and could be multidimensional arrays of the same shape, or a scalar and an array, or even two arrays with different shapes, provided that the smaller array is “expandable” to the shape of the larger in such a way that the resulting broadcast is unambiguous. For detailed “rules” of broadcasting see Broadcasting.ab

Who else uses NumPy?#

NumPy fully supports an object-oriented approach, starting, once again, with ndarray. For example, ndarray is a class, possessing numerous methods and attributes. Many of its methods are mirrored by functions in the outer-most NumPy namespace, allowing the programmer to code in whichever paradigm they prefer. This flexibility has allowed the NumPy array dialect and NumPy ndarray class to become the de-facto language of multi-dimensional data interchange used in Python.
