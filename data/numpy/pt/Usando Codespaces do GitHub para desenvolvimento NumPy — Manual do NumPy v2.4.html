Usando Codespaces do GitHub para desenvolvimento em NumPy#

Esta seção da documentação irá guiá-lo por:

  • usando GitHub Codespaces para seu ambiente de desenvolvimento NumPy

  • criando um fork pessoal do repositório NumPy no GitHub

  • um rápido tour pelos Codespaces do GitHub e pelo aplicativo de desktop VSCode

  • trabalhando na documentação do NumPy no GitHub Codespaces

GitHub Codespaces#

GitHub Codespaces é um serviço que oferece baseado em nuvem ambientes de desenvolvimento para que você não precise instalar nada Na sua máquina local ou se preocupe com a configuração.

O que é um espaço de código?#

Um espaço de código é uma instância de Espaços de Código - e, portanto, um ambiente de desenvolvimento que é hospedado na nuvem. Cada espaço de código roda em uma máquina virtual hospedada por GitHub. Você pode escolher o tipo de máquina que deseja usar, dependendo do Recursos que você precisa. Vários tipos de máquinas estão disponíveis, começando por um Processador de 2 núcleos, 4 GB de RAM e 32 GB de armazenamento. Você pode se conectar a um código do seu navegador, do Visual Studio Code, do JetBrains Aplicativo gateway, ou usando a CLI do GitHub.

Forkando o repositório NumPy#

A melhor forma de trabalhar na base de código do NumPy como contribuinte é criando um fork primeiro do repositório.

  1. Navegue até o repositório NumPy no GitHub e crie seu próprio fork.

  2. Navegue até o seu garfo de árvores. Seu fork terá uma URL como inessapawson/numpy, mas com seu nome de usuário no GitHub no lugar de .inessapawson

Iniciando o GitHub Codespaces#

Você pode criar um espaço de código a partir do botão verde "<> Code" no repositório Página inicial e escolha "Codespaces", ou clique neste link para abrir.

Visita rápida ao espaço de trabalho#

Você pode desenvolver código em um espaço de código usando a ferramenta que escolher:

  • um shell de comando, via uma conexão SSH iniciada usando o GitHub CLI._

  • um dos IDEs do JetBrains, via o Gateway._ JetBrains

  • uma versão baseada em navegador do Visual Studio Code._

FAQs and troubleshooting#

How long does my codespace stay active if I’m not using it? If you leave your codespace running without interaction, or if you exit your codespace without explicitly stopping it, by default the codespace will timeout after 30 minutes of inactivity. You can customize the duration of the timeout period for new codespaces that you create.

Can I come back to a previous codespace? The lifecycle of a codespace begins when you create a codespace and ends when you delete it. You can disconnect and reconnect to an active codespace without affecting its running processes. You may stop and restart a codespace without losing changes that you have made to your project.
