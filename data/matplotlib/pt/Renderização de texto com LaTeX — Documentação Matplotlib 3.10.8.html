Renderização de texto com LaTeX#

O Matplotlib pode usar LaTeX para renderizar texto. Isso é ativado ao definir seus rcParams, ou ao definir a propriedade até Verdadeiro em objetos individuais. O tratamento de texto pelo LaTeX é mais lento do que o texto matemático muito competente de Matplotlib, mas é mais flexível, já que diferentes pacotes LaTeX (pacotes de fontes, pacotes de matemática, etc.) pode ser usado. Os resultados podem ser marcantes, especialmente quando você cuida Usar as mesmas fontes nas suas figuras que no documento principal.text.usetex : Trueusetex

O suporte a LaTeX da Matplotlib requer uma instalação LaTeX funcionando. Para os backends *Agg, dvipng também são necessários; para o backend do PS, PSfrag, DVIPs e Ghostscript também são necessários. Para o PDF e backends SVG, se o LuaTeX estiver presente, será usado para acelerar um pouco passos de pós-processamento, mas note que não é usado para analisar a string TeX ela mesma (apenas LaTeX é suportado). Os executáveis desses externos dependências devem estar todas localizadas em seus arquivos .

Apenas um pequeno número de famílias de fontes (definidas pelo esquema PSNFSS) são apoiado. Elas estão listadas aqui, com a seleção correspondente de fontes LaTeX comandos e pacotes LaTeX, que são usados automaticamente.

A família de fontes padrão (que não exige carregamento de nenhum pacote LaTeX) é Computador Moderno. Todas as outras famílias são fontes Adobe. Times e Palatino cada possuem suas próprias fontes matemáticas associadas, enquanto as outras fontes serifais da Adobe fazem uso das fontes matemáticas Computer Modern.

Para habilitar o LaTeX e selecionar uma fonte, use, por exemplo:

plt.rcParams.update({
    "text.usetex": True,
    "font.family": "Helvetica"
})

Ou, equivalentemente, defina seu matplotlibrc para:

text.usetex : true
font.family : Helvetica

Também é possível definir para um da família genérica nomes e então configuram a família genérica correspondente; Por exemplo:font.family

plt.rcParams.update({
    "text.usetex": True,
    "font.family": "sans-serif",
    "font.sans-serif": "Helvetica",
})

(essa era a abordagem necessária até Matplotlib 3.5).

PostScript options#

In order to produce encapsulated PostScript (EPS) files that can be embedded in a new LaTeX document, the default behavior of Matplotlib is to distill the output, which removes some PostScript operators used by LaTeX that are illegal in an EPS file. This step produces results which may be unacceptable to some users, because the text is coarsely rasterized and converted to bitmaps, which are not scalable like standard PostScript, and the text is not searchable. One workaround is to set (default: ) to a higher value (perhaps 6000) in your rc settings, which will produce larger files but may look better and scale reasonably. A better workaround, which requires Poppler or Xpdf, can be activated by changing (default: ) to . This alternative produces PostScript without rasterizing text, so it scales properly, can be edited in Adobe Illustrator, and searched text in pdf documents.rcParams["ps.distiller.res"]6000rcParams["ps.usedistiller"]Nonexpdf

Possible hangups#

  • On Windows, the PATH environment variable may need to be modified to include the directories containing the latex, dvipng and ghostscript executables. See Environment variables and Setting environment variables in Windows for details.

  • Using MiKTeX with Computer Modern fonts, if you get odd *Agg and PNG results, go to MiKTeX/Options and update your format files.

  • On Ubuntu and Gentoo, the base texlive install does not ship with the type1cm package. You may need to install some of the extra packages to get all the goodies that come bundled with other LaTeX distributions.

Troubleshooting#

  • Try deleting your directory. If you don't know where to find , see matplotlib configuration and cache directory locations..matplotlib/tex.cache.matplotlib

  • Make sure LaTeX, dvipng and Ghostscript are each working and on your PATH.

  • Make sure what you are trying to do is possible in a LaTeX document, that your LaTeX syntax is valid and that you are using raw strings if necessary to avoid unintended escape sequences.

  • rcParams["text.latex.preamble"] (default: ) is not officially supported. This option provides lots of flexibility, and lots of ways to cause problems. Please disable this option before reporting problems.''

  • If you still need help, please see Get help.
