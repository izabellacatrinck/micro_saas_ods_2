Renderização de texto com XeLaTeX/LuaLaTeX via backendpgf#

Usando o backend, o Matplotlib pode exportar figuras como desenho pgf comandos que podem ser processados com PDFLATEX, Xelatex ou Lualatex. XeLaTeX e O LuaLaTeX tem suporte total ao Unicode e pode usar qualquer fonte instalada em o sistema operacional, utilizando recursos tipográficos avançados do OpenType, AAT e grafite. Imagens Pgf criadas por podem ser incorporadas como comandos raw em documentos LaTeX. Figuras também podem ser compilado diretamente e salvo em PDF com Trocando o backendpgfplt.savefig('figure.pgf')plt.savefig('figure.pdf')

matplotlib.use('pgf')

ou solicitando explicitamente o uso do backendpgf

plt.savefig('figure.pdf', backend='pgf')

ou registrando-o para lidar com a saída do PDF

from matplotlib.backends.backend_pgf import FigureCanvasPgf
matplotlib.backend_bases.register_backend('pdf', FigureCanvasPgf)

O último método permite que você continue usando backends interativos regulares e para Salve arquivos PDF compilados Xelatex, Lualatex ou PDFLATEX do usuário gráfico interface. Note que, nesse caso, a tela interativa ainda usará o backends interativos padrão (por exemplo, QtAgg), e em particular usam latex para Compile trechos de texto relevantes.

O suporte a pgf da Matplotlib exige uma instalação recente do LaTeX que inclui os pacotes TikZ/PGF (como o TeXLive), preferencialmente com XeLaTeX ou LuaLaTeX instalado. Se pdftocairo ou ghostscript estiver presente no seu sistema, figuras também podem ser salvas opcionalmente em imagens PNG. Os executáveis Para todas as aplicações, deve estar localizado em seu arquivo .

Nota

O TeX define um conjunto de caracteres especiais, tais como:

# $ % & ~ _ ^ \ { }

Geralmente, esses personagens devem ser escapados corretamente. Para conveniência, alguns caracteres (_, ^, %) são automaticamente escapados fora da matemática ambientes. Outros personagens não escapam como são comumente necessários em expressões reais de TeX. No entanto, pode-se configurar o TeX para tratá-los como caracteres "normais" (conhecidos como "catcode 12" para o TeX) via um preâmbulo personalizado, como:

plt.rcParams["pgf.preamble"] = (
    r"\AtBeginDocument{\catcode`\&=12\catcode`\#=12}")

Arquivos PDF de Várias Páginas#

O backend do pgf também suporta arquivos pdf multipágina usando 

from matplotlib.backends.backend_pgf import PdfPages
import matplotlib.pyplot as plt

with PdfPages('multipage.pdf', metadata={'author': 'Me'}) as pdf:

    fig1, ax1 = plt.subplots()
    ax1.plot([1, 5, 3])
    pdf.savefig(fig1)

    fig2, ax2 = plt.subplots()
    ax2.plot([1, 5, 3])
    pdf.savefig(fig2)

Custom preamble#

Full customization is possible by adding your own commands to the preamble. Use (default: ) if you want to configure the math fonts, using for example, or for loading additional packages. Also, if you want to do the font configuration yourself instead of using the fonts specified in the rc parameters, make sure to disable (default: ).rcParams["pgf.preamble"]''unicode-mathrcParams["pgf.rcfonts"]True

import matplotlib as mpl

mpl.use("pgf")
import matplotlib.pyplot as plt

plt.rcParams.update({
    "font.family": "serif",  # use serif/main font for text elements
    "text.usetex": True,     # use inline math for ticks
    "pgf.rcfonts": False,    # don't setup fonts from rc parameters
    "pgf.preamble": "\n".join([
         r"\usepackage{url}",            # load additional packages
         r"\usepackage{unicode-math}",   # unicode math setup
         r"\setmainfont{DejaVu Serif}",  # serif font via preamble
    ])
})

fig, ax = plt.subplots(figsize=(4.5, 2.5))

ax.plot(range(5))

ax.set_xlabel("unicode text: я, ψ, €, ü")
ax.set_ylabel(r"\url{https://matplotlib.org}")
ax.legend(["unicode math: $λ=∑_i^∞ μ_i^2$"])

Troubleshooting#

  • On Windows, the PATH environment variable may need to be modified to include the directories containing the latex, dvipng and ghostscript executables. See Environment variables and Setting environment variables in Windows for details.

  • Sometimes the font rendering in figures that are saved to png images is very bad. This happens when the pdftocairo tool is not available and ghostscript is used for the pdf to png conversion.

  • Make sure what you are trying to do is possible in a LaTeX document, that your LaTeX syntax is valid and that you are using raw strings if necessary to avoid unintended escape sequences.

  • rcParams["pgf.preamble"] (default: ) provides lots of flexibility, and lots of ways to cause problems. When experiencing problems, try to minimalize or disable the custom preamble.''

  • Configuring an environment can be a bit tricky. The TeXLive distribution for example provides a set of math fonts which are usually not installed system-wide. XeLaTeX, unlike LuaLaTeX, cannot find these fonts by their name, which is why you might have to specify instead of or alternatively make the fonts available to your OS. See this tex.stackexchange.com question for more details.unicode-math\setmathfont{xits-math.otf}\setmathfont{XITS Math}

  • If the font configuration used by Matplotlib differs from the font setting in yout LaTeX document, the alignment of text elements in imported figures may be off. Check the header of your file if you are unsure about the fonts Matplotlib used for the layout..pgf

  • Vector images and hence files can become bloated if there are a lot of objects in the graph. This can be the case for image processing or very big scatter graphs. In an extreme case this can cause TeX to run out of memory: "TeX capacity exceeded, sorry" You can configure latex to increase the amount of memory available to generate the image as discussed on tex.stackexchange.com. Another way would be to "rasterize" parts of the graph causing problems using either the keyword, or as per this example..pgf.pdfrasterized=True.set_rasterized(True)

  • Various math fonts are compiled and rendered only if corresponding font packages are loaded. Specifically, when using on Greek letters, the default computer modern font may not contain them, in which case the letter is not rendered. In such scenarios, the package should be loaded.\mathbf{}lmodern

  • If you still need help, please see Get help
