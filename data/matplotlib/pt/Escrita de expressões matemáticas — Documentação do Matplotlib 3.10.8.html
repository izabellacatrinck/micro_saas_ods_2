Escrevendo expressões matemáticas#

O Matplotlib implementa um parser de expressões e um motor de layout leve para TeX, e o Mathtext é o subconjunto da marcação Tex que este motor suporta. Note que O Matplotlib também pode renderizar todo o texto diretamente usando TeX se (padrão: ) for Verdadeiro; veja Renderização de texto com LaTeX para mais detalhes. O suporte ao Mathtext está disponível se (padrão: ) for Falso.rcParams["text.usetex"]FalsercParams["text.usetex"]False

Qualquer string pode ser processada como Mathtext colocando-a dentro de um par de Cifrões. O texto matemático frequentemente contém muitas barras adicionais; de modo que as barras inversas não precisam ser evitadas, o Mathtext geralmente é escrito usando raw cordas. Por exemplo:'$''\'

import matplotlib.pyplot as plt

fig = plt.figure(figsize=(3, 3), linewidth=1, edgecolor='black')
fig.text(.2, .7, "plain text: alpha > beta")
fig.text(.2, .5, "Mathtext: $\\alpha > \\beta$")
fig.text(.2, .3, r"raw string Mathtext: $\alpha > \beta$")

Personagens especiais#

O texto matemático deve ser colocado entre um par de cifrões (EUA). Um literal o símbolo do dólar em uma string contendo Mathtext deve ser escapado usando um Barra inversa: . Uma corda pode conter vários pares de cifrões, resultando em múltiplas expressões Mathtext. Cordas com número ímpar de Os cifrões são apresentados apenas como texto simples.'$''$''\$'

fig = plt.figure(figsize=(3, 3), linewidth=1, edgecolor='black')
fig.suptitle("Number of unescaped $")
fig.text(.1, .7, r"odd: $ \alpha $ = $1")
fig.text(.1, .5, r"even: $ \beta $= $ 2 $")
fig.text(.1, .3, r'odd: $ \gamma $= \$3 $')
fig.text(.1, .1, r'even: $ \delta $ = $ \$4 $')

Embora o Mathtext busque compatibilidade com o TeX comum, ele diverge quanto a quando Personagens especiais precisam ser escapados. No TeX, o cifrão deve ser evitado em textos não matemáticos, enquanto no Matplotlib o cifrão deve ser escapou enquanto escrevia Mathtext.'\$'

Esses outros personagens especiais também escapam em TeX não-matemático, enquanto em Matplotlib seu comportamento depende de como (padrão: ) é definido:rcParams["text.usetex"]False

Subscritos e subscritos#

Para criar subscritos e súbitos, use os símbolos e e:'_''^'

r'$\alpha_i > \beta_i$'
αi>βi

To display multi-letter subscripts or superscripts correctly, you should put them in curly braces :{...}

r'$\alpha^{ic} > \beta_{ic}$'
αic>βic

Some symbols automatically put their sub/superscripts under and over the operator. For example, to write the sum of from to , you could do:

r'$\sum_{i=0}^\infty x_i$'

Fractions, binomials, and stacked numbers#

Fractions, binomials, and stacked numbers can be created with the , and commands, respectively:\frac{}{}\binom{}{}\genfrac{}{}{}{}{}{}

r'$\frac{3}{4} \binom{3}{4} \genfrac{}{}{0}{}{3}{4}$'

produces

34(34)34

Fractions can be arbitrarily nested:

r'$\frac{5 - \frac{1}{x}}{4}$'

produces

5−1x4

Note that special care needs to be taken to place parentheses and brackets around fractions. Doing things the obvious way produces brackets that are too small:

r'$(\frac{5 - \frac{1}{x}}{4})$'
(5−1x4)

The solution is to precede the bracket with and to inform the parser that those brackets encompass the entire object.:\left\right

r'$\left(\frac{5 - \frac{1}{x}}{4}\right)$'
(5−1x4)

Radicals#

Radicals can be produced with the command. For example:\sqrt[]{}

r'$\sqrt{2}$'
2

Any base can (optionally) be provided inside square brackets. Note that the base must be a simple expression, and cannot contain layout commands such as fractions or sub/superscripts:

r'$\sqrt[3]{x}$'
x3

Fonts#

The default font is italics for mathematical symbols.

This default can be changed using (default: ). For setting rcParams, see Customizing Matplotlib with style sheets and rcParams. For example, setting the default to allows you to use the same font for math text and regular non-math text.rcParams["mathtext.default"]'it'regular

To change fonts, e.g., to write "sin" in a Roman font, enclose the text in a font command:

r'$s(t) = \mathcal{A}\mathrm{sin}(2 \omega t)$'
s(t)=Asin(2ωt)

More conveniently, many commonly used function names that are typeset in a Roman font have shortcuts. So the expression above could be written as follows:

r'$s(t) = \mathcal{A}\sin(2 \omega t)$'
s(t)=Asin⁡(2ωt)

Here "s" and "t" are variable in italics font (default), "sin" is in Roman font, and the amplitude "A" is in calligraphy font. Note in the example above the calligraphy is squished into the . You can use a spacing command to add a little whitespace between them:Asin

r's(t) = \mathcal{A}\/\sin(2 \omega t)'
s(t)=Asin⁡(2ωt)
