Backends#

O que é um backend?#

Backends são usados para exibir figuras Matplotlib (veja Introdução às Figuras), na tela, ou para escrever em arquivos. Muita documentação no site e nas listas de discussão refere-se ao "backend" e muitos novos usuários são confuso com esse termo. Matplotlib mira em muitos casos de uso e resultados diferentes formatos. Algumas pessoas usam o Matplotlib interativamente a partir do shell Python e Faça janelas de plotagem aparecerem quando eles digitarem comandos. Algumas pessoas usam cadernos Jupyter e desenham gráficos em linha para dados rápidos análise. Outros incorporam o Matplotlib em interfaces gráficas como PyQt ou PyGObject para construir aplicações ricas. Algumas pessoas usam Matplotlib em lote scripts para gerar imagens pós-script a partir de simulações numéricas, e ainda outros executam servidores de aplicações web para fornecer gráficos dinamicamente.

Para suportar todos esses casos de uso, o Matplotlib pode direcionar diferentes tipos saídas, e cada uma dessas capacidades é chamada de backend; o "frontend" é o código voltado para o usuário, ou seja, o código de plotamento, enquanto o "Backend" faz todo o trabalho duro nos bastidores para criar a figura. Existem dois tipos de backends: backends de interface de usuário (para uso em PyQt/PySide, PyGObject, Tkinter, wxPython ou macOS/Cocoa); também referido como "backends interativos") e backends físicos para criar arquivos de imagem (PNG, SVG, PDF, PS; também chamado de "backends não interativos").

Seleção de um backend#

Existem três maneiras de configurar seu backend:

  • O parâmetro no seu arquivorcParams["backend"]matplotlibrc

  • A variável ambiente

  • A função

Abaixo está uma descrição mais detalhada.

Se houver mais de uma configuração presente, a última do A lista tem prioridade; por exemplo, chamar a substituição A configuração em seu .matplotlibrc

Sem um backend explicitamente definido, o Matplotlib detecta automaticamente um utilizável backend baseado no que está disponível no seu sistema e em se um evento de interface gráfica Loop já está rodando. O primeiro backend utilizável na lista a seguir é selecionados: MacOSX, QtAgg, GTK4Agg, Gtk3Agg, TkAgg, WxAgg, Agg. A última, Agg, é um backend não interativo que só pode gravar em arquivos. É usado em Linux, se o Matplotlib não consegue se conectar nem a um monitor X nem a um Wayland display.

Aqui está uma descrição detalhada dos métodos de configuração:

  1. Configuração no seu arquivo:rcParams["backend"]matplotlibrc

    backend : qtagg   # use pyqt with antigrain (agg) rendering

    Veja também Personalização do Matplotlib com folhas de estilo e rcParams.

  2. Definindo a variável de ambiente:

    Você pode definir a variável ambiente tanto para sua concha atual quanto para um único roteiro.

    Sobre Unix:

    > export MPLBACKEND=qtagg
> python simple_plot.py

> MPLBACKEND=qtagg python simple_plot.py

    On Windows, only the former is possible:

    > set MPLBACKEND=qtagg
> python simple_plot.py

    Setting this environment variable will override the parameter in any , even if there is a in your current working directory. Therefore, setting MPLBACKEND globally, e.g. in your or , is discouraged as it might lead to counter-intuitive behavior.backendmatplotlibrcmatplotlibrc.bashrc.profile

  3. If your script depends on a specific backend you can use the function matplotlib.use():

    import matplotlib
matplotlib.use('qtagg')

    This should be done before any figure is created, otherwise Matplotlib may fail to switch the backend and raise an ImportError.

    Using use will require changes in your code if users want to use a different backend. Therefore, you should avoid explicitly calling use unless absolutely necessary.

The builtin backends#

By default, Matplotlib should automatically select a default backend which allows both interactive work and plotting from scripts, with output to the screen and/or to a file, so at least initially, you will not need to worry about the backend. The most common exception is if your Python distribution comes without tkinter and you have no other GUI toolkit installed. This happens with certain Linux distributions, where you need to install a Linux package named (or similar).python-tk

If, however, you want to write graphical user interfaces, or a web application server (Embed in a web application server (Flask)), or need a better understanding of what is going on, read on. To make things easily more customizable for graphical user interfaces, Matplotlib separates the concept of the renderer (the thing that actually does the drawing) from the canvas (the place where the drawing goes). The canonical renderer for user interfaces is which uses the Anti-Grain Geometry C++ library to make a raster (pixel) image of the figure; it is used by the , , , , , and backends. An alternative renderer is based on the Cairo library, used by , etc.AggQtAggGTK4AggGTK3AggwxAggTkAggmacosxQtCairo

For the rendering engines, users can also distinguish between vector or raster renderers. Vector graphics languages issue drawing commands like "draw a line from this point to this point" and hence are scale free. Raster backends generate a pixel representation of the line whose accuracy depends on a DPI setting.

ipympl#

The ipympl backend is in a separate package that must be explicitly installed if you wish to use it, for example:

pip install ipympl

or

conda install ipympl -c conda-forge

See installing ipympl for more details.

Using non-builtin backends#

More generally, any importable backend can be selected by using any of the methods above. If is the module containing the backend, use as the backend name, e.g. .name.of.the.backendmodule://name.of.the.backendmatplotlib.use('module://name.of.the.backend')

Information for backend implementers is available at Writing a backend -- the pyplot interface.

Debugging the figure windows not showing#

Sometimes things do not work as expected, usually during an install.

If you are using a Notebook or integrated development environment (see Notebooks and IDEs), please consult their documentation for debugging figures not working in their environments.

If you are using one of Matplotlib's graphics backends (see Standalone scripts and interactive use), make sure you know which one is being used:

import matplotlib

print(matplotlib.get_backend())

Try a simple plot to see if the GUI opens:

import matplotlib
import matplotlib.pyplot as plt

print(matplotlib.get_backend())
plt.plot((1, 4, 6))
plt.show()

If it does not, you perhaps have an installation problem. A good step at this point is to ensure that your GUI toolkit is installed properly, taking Matplotlib out of the testing. Almost all GUI toolkits have a small test program that can be run to test basic functionality. If this test fails, try re-installing.
