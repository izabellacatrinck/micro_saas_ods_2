Tipos de dados de texto#

Existem duas maneiras de armazenar dados de texto em pandas:

  1. Tipo de extensão.

  2. Dtype Anestesiado.object

Recomendamos usar para armazenar dados de texto via o alias (o padrão quando dtype de cadeias é inferido), veja abaixo para mais detalhes.dtype="str"

Antes do pandas 1.0, dtype era a única opção. Isso foi lamentável Por vários motivos:object

  1. Você pode armazenar acidentalmente uma mistura de strings e não-strings em um array dtype. É melhor ter um dtype dedicado.object

  2. object dtype quebra operações específicas de dtype como . Não há uma maneira clara de selecionar apenas texto excluindo o não-texto mas ainda assim são colunas de type object-dtype.

  3. Ao ler código, o conteúdo de um array dtype é menos claro que .object'string'

Ao usar com PyArrow como armazenamento (veja abaixo), Os usuários verão grandes melhorias de desempenho tanto na memória quanto no tempo para certas operações quando comparadas a matrizes dtype. Quando não usando o PyArrow como armazenamento, o desempenho de é aproximadamente o mesmo de . Esperamos melhorias futuras para aumentar significativamente o desempenho e diminuir a sobrecarga de memória de neste caso.objectobject
