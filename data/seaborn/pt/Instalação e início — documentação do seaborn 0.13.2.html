Instalação e início#

Versões oficiais do seaborn podem ser instaladas pelo PyPI:

pip install seaborn

A invocação básica de will instala seaborn e, se necessário, suas dependências obrigatórias. É possível incluir dependências opcionais que dão acesso a alguns recursos avançados:pip

pip install seaborn[stats]

A biblioteca também está incluída como parte da distribuição da Anaconda, e pode ser instalado com:conda

conda install seaborn

Como o repositório principal da Anaconda pode demorar para adicionar novos lançamentos, você pode preferir usar o canal conda-forge:

conda install seaborn -c conda-forge

Dependências#

Versões suportadas em Python#

  • Python 3.8+

Dependências obrigatórias#

  • numpy

  • Pandas

  • matplotlib

Dependências opcionais#

  • Statsmodels, para gráficos avançados de regressão

  • scipy, para clustering de matrizes e algumas opções avançadas

  • fastcluster, clustering mais rápido de matrizes grandes

Início rápido#

Depois de instalar o Seaborn, estará pronto para começar. Para testar, você poderia carregar e plotar um dos conjuntos de dados de exemplo:

import seaborn as sns
df = sns.load_dataset("penguins")
sns.pairplot(df, hue="species")

Se você estiver trabalhando em um notebook Jupyter ou em um terminal IPython com o modo matplotlib ativado, deve ver imediatamente o gráfico. Caso contrário, talvez seja necessário chamar explicitamente :

import matplotlib.pyplot as plt
plt.show()

Embora você possa ir bem longe apenas com Seaborn importado, tendo acesso a As funções Matplotlib costumam ser úteis. Os tutoriais e a documentação da API Normalmente assumem as seguintes importações:

import numpy as np
import pandas as pd

import matplotlib as mpl
import matplotlib.pyplot as plt

import seaborn as sns
import seaborn.objects as so

Problemas de instalação de depuração#

O código do seaborn é puro Python, e a biblioteca geralmente deve ser instalada Sem descendência. Ocasionalmente, surgem dificuldades devido às dependências inclua código compilado e link para bibliotecas do sistema. Essas dificuldades normalmente se manifestam como erros na importação com mensagens como . Para depurar tais problemas, leia o rastreamento de exceção para Descubra qual biblioteca específica falhou em importar e então consulte o Documentação de instalação desse pacote para ver se eles têm dicas para o seu caso específico sistema."DLL load failed"

Em alguns casos, uma instalação de seaborn parecerá ter sucesso, mas está tentando para importá-la gerará um erro com a mensagem . Isso geralmente significa que você tem várias instalações de Python em seu sistema e que seu ou aponta para um sistema diferente Instalação do que onde seu intérprete mora. Buscando ajuda#

Se você acha que encontrou um bug no seaborn, por favor, reporte no rastreador de problemas do GitHub. Para serem úteis, os relatórios de bugs devem incluir as seguintes informações:

  • Um exemplo de código reproduzível que demonstra o problema

  • A saída que você está vendo (uma imagem de um gráfico ou a mensagem de erro)

  • Uma explicação clara do porquê você acha que algo está errado

  • As versões específicas de seaborn e matplotlib com as quais você está trabalhando

Relatórios de bugs são mais fáceis de resolver se puderem ser demonstrados usando um dos Exemplos de conjuntos de dados do Seaborn Docs (ou seja, com ). Caso contrário, é preferível que seu exemplo gere dados sintéticos para Reproduza o problema. Se você só puder demonstrar o problema com seu Conjunto de dados propriamente ditos, você precisará compartilhá-lo, idealmente como um CSV.

Se você encontrou um erro, procure o texto específico da mensagem Antes de abrir uma nova edição, muitas vezes você pode ajudar a resolvê-lo rapidamente e Evite fazer um relatório duplicado.

Porque o matplotlib cuida da renderização real, erros ou resultados incorretos Pode ser devido a um problema no Matplotlib em vez de um no Seaborn. Isso pode economizar tempo Se você tentar reproduzir o problema em um exemplo que use apenas o matplotlib, Para que você possa denunciar no lugar certo. Mas tudo bem pular isso Passe se não estiver óbvio como fazer.

Perguntas de suporte geral são mais acessíveis no stackoverflow, onde há um Público maior de pessoas que verão seu post e podem oferecer Assistência. Sua chance de obter uma resposta rápida será maior se você incluir código executável, uma declaração precisa do que você espera alcançar e uma explicação clara dos problemas que você encontrou.
