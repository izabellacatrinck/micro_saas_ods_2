Propriedades dos objetos Mark#

Propriedades das coordenadas#

x, y, xmin, xmax, ymin, ymax#

As propriedades das coordenadas determinam onde uma marca é desenhada em um gráfico. Canônicamente, a coordenada é a posição horizontal e a coordenada é a posição vertical. Algumas marcas aceitam uma parametrização por intervalo (ou seja, , ) para uma ou ambas as variáveis. Outros podem aceitar e também usar um parâmetro para mostrar um intervalo. O parâmetro da camada determina como isso funciona.xyminmaxxybaselineorient

Se uma variável não contiver dados numéricos, sua escala aplicará uma conversão para que os dados possam ser desenhados em uma tela. Por exemplo, escalas atribuem um índice inteiro a cada categoria distinta, e escalas representam datas como o número de dias a partir de uma "época" de referência:NominalTemporal

Uma escala também pode aplicar uma transformação não linear entre valores de dados e posições espaciais:Continuous

Alfa, preenchido, Edgealpha#

A propriedade determina a opacidade da marca. Reduzir o alfa pode ser útil para representar a densidade no caso de sobreplotamento:alpha

Mapping the property can also be useful even when marks do not overlap because it conveys a sense of importance and can be combined with a scale to represent two variables. Moreover, colors with lower alpha appear less saturated, which can improve the appearance of larger filled marks (such as bars).alphacolor

Style properties#

fill#

The property is relevant to marks with a distinction between the edge and interior and determines whether the interior is visible. It is a boolean state: can be set only to or :fillfillTrueFalse

marker#

The property is relevant for dot marks and some line marks. The API for specifying markers is very flexible, as detailed in the matplotlib API docs: matplotlib.markers.marker

Markers can be specified using a number of simple string codes:

They can also be programatically generated using a tuple:(num_sides, fill_style, angle)

A marker property is always mapped with a nominal scale; there is no inherent ordering to the different shapes. If no scale is provided, the plot will programmatically generate a suitably large set of unique markers:

While this ensures that the shapes are technically distinct, bear in mind that — in most cases — it will be difficult to tell the markers apart if more than a handful are used in a single plot.

Note

The default marker scale is subject to change in future releases to improve discriminability.

Size properties#

pointsize#

The property is relevant to dot marks and to line marks that can show markers at individual data points. The units correspond to the diameter of the mark in points.pointsize

Note that, while the parameterization corresponds to diameter, scales will be applied with a square root transform so that data values are linearly proportional to area:

linewidth#

The property is relevant to line marks and determines their thickness. The value should be non-negative and has point units:linewidth

edgewidth#

The property is akin to but applies to marks with an edge/fill rather than to lines. It also has a different default range when used in a scale. The units are the same:edgewidthlinewidth

stroke#

The property is akin to but applies when a dot mark is defined by its stroke rather than its fill. It also has a slightly different default scale range, but otherwise behaves similarly:strokeedgewidth

Text properties#

halign, valign#

The and properties control the horizontal and vertical alignment of text marks. The options for horizontal alignment are , , and , while the options for vertical alignment are , , , , and .halignvalign'left''right''center''top''bottom''center''baseline''center_baseline'

fontsize#

The property controls the size of textual marks. The value has point units:fontsize

offset#

The property controls the spacing between a text mark and its anchor position. It applies when not using alignment (i.e., when using left/right or top/bottom). The value has point units.offsetcenter

Other properties#

text#

The property is used to set the content of a textual mark. It is always used literally (not mapped), and cast to string when necessary.text

group#

The property is special in that it does not change anything about the mark’s appearance but defines additional data subsets that transforms should operate on independently.group
